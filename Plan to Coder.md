# [System Upgrade] 에너미 스폰 물량 대규모 상향 명세서

이 문서는 게임 전반의 에너미 스폰 개수를 조정하여 'Cyber Survivor' 특유의 압도적인 물량 공세를 구현하기 위한 마스터 가이드입니다.

---

## 1. 에너미 스폰 개수 확장 (`engine.js` - spawnEnemy)

스폰 주기(Interval)는 유지하되, 1회 스폰 시 생성되는 개체수를 상향 조정합니다.

### 1.1 일반 스테이지 (Stage 1 & 2)
- **기존**: `spawnEnemy()` 호출 시 에너미 1마리 생성.
- **변경**: `spawnEnemy()` 호출 시 **에너미 2마리**가 생성되도록 루프 처리합니다.
  - 두 마리는 각각 `Utils.getSpawnPos()`를 호출하여 완전히 다른 위치에서 스폰되어야 합니다.
  - 소환되는 2마리 중 최소 1마리는 반드시 `normal` 타입이 되도록 가중치를 부여하세요.

### 1.2 무한 모드 (Infinite Mode)
- **기존**: `spawnCount = 2 ~ 3`마리 무작위 생성.
- **변경**: `spawnCount = 4 ~ 6`마리 무작위 생성으로 값을 **2배** 상향합니다.

### 1.3 보스전 지원군 (Glitch Weaver)
- **변경**: `triggerGlitchMinionSpawn` 시 생성되는 물량을 기존 `8 ~ 12`마리에서 **16 ~ 24마리**로 대폭 상향합니다.

## 2. 시각적 가독성 유지 (`entities.js` - Enemy)

물량이 늘어남에 따라 화면이 혼잡해지는 것을 막기 위해 다음을 적용합니다.
- **Normal Enemy 전용**: 스테이지 2 진입 시 보라색 변이는 `normal` 타입에게만 적용하여 특수 적(Ranged, Bomber 등)과의 시각적 혼돈을 방지합니다.

---

## 3. [Bug Fix] 프리즘 어빌리티 '반전술식' 로직 수정

### 3.1 로직 오류 수정 (`entities.js` - applyUpgrade)
- **현상**: 공격력과 **현재 체력(hp)**이 바뀌어, 부상 상태에서 획득 시 공격력이 급감함.
- **해결**: 공격력(`attackDamage`)과 **최대 체력(maxHp)**을 교환하도록 수정합니다.
- **상세 코드 지침**:
  ```javascript
  const tmp = this.attackDamage;
  this.attackDamage = this.maxHp;
  this.maxHp = tmp;
  this.hp = this.maxHp; // 교체 후 체력을 풀피로 회복
  ```

## ⚠️ [CRITICAL] Coder를 위한 구현 및 성능 지침

> [!IMPORTANT]
> **개체 수 2배 증가는 CPU/GPU 부하를 동반합니다.**

1. **파티클 최적화**: 많은 적이 동시에 죽을 때 발생하는 파티클 개수가 과도해지지 않도록 `createParticles` 호출 시의 파티클 개수를 상황에 따라 소폭 제한하는 것을 고려하세요.
2. **백업 필수**: 물량 증가 후 브라우저 프리징 현상이 발생하는지 테스트하고, 문제 발생 시 즉시 이전 스폰 로직으로 롤백할 수 있도록 원본 코드를 보존하세요.
3. **젬(Gem) 관리**: 적이 2배로 늘어나면 필드의 젬 개수도 2배가 됩니다. 젬이 일정 시간 이상 방치될 경우 병합(Merge)되거나 사라지는 로직이 필요한지 확인하세요.
