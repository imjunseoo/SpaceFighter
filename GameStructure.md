# 🌌 Cyber Survivor: 통합 게임 아키텍처 명세서

이 문서는 'Cyber Survivor' 프로젝트의 코드 구조, 클래스 설계, 그리고 주요 함수들의 상호작용을 정리한 기술 가이드입니다.

---

## 📂 프로젝트 파일 구조

| 파일명 | 역할 |
| :--- | :--- |
| `index.html` | 게임 화면 구성, UI 레이어 및 외부 라이브러리(Firebase 등) 로드 |
| `css/style.css` | 게임 UI 애니메이션, 레이아웃 및 테두리 효과 등 스타일링 |
| `js/config.js` | 게임 밸런스 데이터, 아티팩트/어빌리티 풀 정의 |
| `js/utils.js` | 거리 계산, 각도 계산, 랜덤 범위 등 공통 헬퍼 함수 |
| `js/entities.js` | 플레이어, 에너미, 투사체, 젬 등 게임 내 객체 클래스 정의 |
| `js/engine.js` | 입출력 관리, 스킬 매니저, 메인 게임 루프 및 UI 핸들링 |
| `js/audio.js` | 사운드 이펙트 및 BGM 관리 (Web Audio API 활용) |

## 🛠️ 핵심 클래스 및 유틸리티

### 📌 `Utils` (js/utils.js)
공통적으로 사용되는 범용 수학/로직 함수입니다.
- **`dist(x1, y1, x2, y2)`**: 두 점 사이의 거리를 반환합니다.
- **`angle(x1, y1, x2, y2)`**: 두 점 사이의 각도를 반환합니다.
- **`clamp(val, min, max)`**: 값을 최소/최대 범위 내로 제한합니다.
- **`getSpawnPos(width, height)`**: 화면 밖 가장자리 무작위 스폰 위치를 계산합니다.

### 👤 `Player` (js/entities.js)
플레이어의 상태와 동작을 제어합니다.
- **`update(input)`**: 대시, 이동, 쿨타임 및 블랙홀/버서커 등 특수 상태를 업데이트합니다.
- **`draw(ctx, frameCount)`**: 플레이어 본체 및 특수 아우라(속박의 왕 등)를 렌더링합니다.
- **`takeDamage(amount)`**: 대미지 처리 및 무적 판정, 리커버리 소생 로직을 수행합니다.
- **`applyUpgrade(skillId)`**: 아티팩트 획득/진화 및 어빌리티 적용을 처리합니다.

### 👾 `Enemy` & `GlitchWeaver` (js/entities.js)
일반 적과 보스의 로직을 관장합니다.
- **`update(player, game)`**: 플레이어 추적, 타입별 이동 패턴(원거리 도망 등) 및 패턴 쿨타임을 갱신합니다.
- **`draw(ctx)`**: 적의 외형과 HP 바(보스 전용)를 렌더링합니다.
- **`GlitchWeaver`**: 2차 보스로서 젬 리버스, 글리치 케이지 등 고유 패턴을 보유합니다.

### 🚀 `Artifact` & `Projectile` 시스템
- **`ArtifactDrone`**: 플레이어 주변을 공전하며 자동으로 공격하는 드론 객체입니다.
- **`DroneMissile`**: 유도(Homing) 또는 직선형 미사일 투사체를 제어합니다.
- **`Bullet`**: 무기나 도구에서 발사되는 기본 투사체입니다. (관통/범위 피해 지원)

## ⚙️ 게임 엔진 및 매니저 (js/engine.js)

### 🎮 `Game` 클래스
전체 게임의 생명주기와 시스템을 총괄하는 컨트롤러입니다.
- **`init()`**: 게임 상태 초기화 및 무기 선택 창을 실행합니다.
- **`loop(timestamp)`**: 메인 루프로서 이동, 충돌, 렌더링, 스폰 등을 매 프레임 실행합니다.
- **`spawnEnemy()`**: 플레이어 레벨 및 상태에 따른 적/보스 스폰을 관리합니다.
- **`showLevelUpMenu()`**: 카드 선택 창을 띄우고 아티팩트/어빌리티 등장 확률을 조절합니다.
- **`processHit(enemy, damage, isCrit)`**: 적의 피격 대미지, 크리티컬, 처치 시 성장 로직을 처리합니다.

### 🪄 `SkillManager` 클래스
시각적 효과와 타격 판정을 관리합니다.
- **`pushMeleeAttack(...)`**: 근접 공격 판정 영역을 생성합니다.
- **`createParticles(x, y, color)`**: 화려한 폭발 파티클 효과를 생성합니다.
- **`updateAndDraw(ctx)`**: 활성화된 모든 공격 이펙트와 파티클, 데미지 숫자를 렌더링합니다.

## ⚔️ 무기 아티팩트 진화 시스템 (Tier 1~4)

Cyber Survivor의 핵심 성장 동력은 아티팩트의 중첩과 진화에 있습니다.

### 1. 진화 알고리즘
- **획득 및 중첩**: 카드 선택 시 동일한 무기 아티팩트를 선택하면 `stacks`가 증가합니다.
- **티어 승급 조건**:
  - **Tier 1**: 최초 획득 (1스택)
  - **Tier 2**: 5스택 달성 시 진화
  - **Tier 3**: 10스택 달성 시 진화
  - **Tier 4 (Final)**: 15스택 달성 시 최종 진화
- **진화 연출**: `engine.js`의 `state = 'EVOLVING'`을 통해 게임이 1.2초간 일시정지되며 전용 이펙트가 출력됩니다.

### 2. 무기별 티어 상세 스펙
| 무기명 | Tier 3 (상급) | Tier 4 (궁극) |
| :--- | :--- | :--- |
| **채찍 (Whip)** | **지배**: X자 형태의 광역 참격 공격 | **소멸**: X자 잔상이 남아서 지속 딜링 후 대규모 폭발 |
| **검 (Sword)** | **아론다이트**: 2단 십자가 콤보 베기 | **엑스칼리버**: 십자가 베기 후 화면 전체 베기 (진동 효과) |
| **권총 (Pistol)** | **협상**: 고속 유성 탄환 자동 추적 사격 | **응징**: 관통 레이저 사격 및 궤적 주변 추가 폭발 |

## 🕹️ 전투 및 공격 메커니즘

### 1. Z키 공격 처리 (`engine.js`)
플레이어의 `weaponArtifact` 종류와 티어에 따라 `triggerAttack` 계열 함수가 호출됩니다.
- **`triggerXShape(atk)`**: 채찍 T3의 X자 공격 판정을 계산합니다.
- **`triggerPiercingLaser(atk)`**: 권총 T4의 직선 관통 레이저와 주변 AoE 데미지를 동시 처리합니다.
- **`triggerCrossUltimate(atk)`**: 검 T4의 3단 콤보를 관리하며, 마지막 타격 시 `screenShake`를 트리거합니다.

### 2. 도구(Tool) 및 드론(Drone) 자동화
- **도구**: `player.toolMultishot` 수치에 따라 `spread` 각도를 계산하여 부채꼴로 여러 발의 투사체를 자동 발사합니다.
- **드론**: `military_drone`으로 진화 시 단일 유도가 아닌 '융단 포격(Carpet Bombing)' 모드로 전환되어 다수의 적을 동시 타격합니다.

## 💾 시스템 및 데이터 저장

### 1. 세이프 익지트 (Safe Exit)
- **메커니즘**: 일시정지 메뉴의 `restart-btn` 클릭 시 단순히 종료하지 않고, `this.player.hp = 0`을 강제 설정한 뒤 `triggerGameOver()`를 호출합니다.
- **데이터 흐름**: `triggerGameOver` -> `deathTimer` 카운트다운 -> `firebaseDB.updateHighScore` 발동.
- **효과**: 플레이어가 중도에 로비로 돌아가더라도 현재까지의 킬수와 생존 기록이 랭킹 시스템에 안전하게 보존됩니다.

### 2. 에너미 스케일링 (Scaling)
- **일반 스테이지**: 플레이어 레벨(Lv)에 선형적으로 비례하여 HP와 속도가 증가합니다.
- **무한 모드**: `infiniteElapsedMinutes`를 기반으로 분당 `1.15^minutes`의 비율(지수 함수)로 적이 강력해집니다.
